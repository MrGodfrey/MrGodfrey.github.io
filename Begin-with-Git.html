Begin with Git

2019-01-30 | 2019-02-03 | 计算机 ， 工具

Git 基础，使用Git管理文件版本。

Git 是一个被广泛使用的版本控制系统，它可以将工作区域（Repository）内所有被追踪的文件的所有修改保留下来，对多人协作有着原生的支持。本文简要总结Git最重要的几个概念和操作，以供日后参考。

Git Basic Ideas

Snapshots

Git 将数据看成是一系列快照，每一次提交，Git都将对当前的文件系统进行一次快照。

The Three States

Git中的文件主要有三种状态，committed,modified 和 staged.

  1. Committed 说明数据已经安全的保存在本地的数据库中
  2. Modified 表示文件被修改，但是还没有提交到本地的数据库中
  3. Staged 表示把修改后的文件当前的状态做个标记，表示它将作为下一次提交的快照的一部分

对应三种状态，Git项目有三个部分，分别是 Git directory (Repository), the working directory(Tree), 以及 staging area.

Git的基本工作流程如下:

  1. 在working tree中修改文件
  2. 暂存(stage)这些修改中将要成为下次提交的部分
  3. 提交(commit)， 将对在staging area和储存中的文件创建Snapshot，并永远保存在Git directory.

The Command Line

Git 主要以命令行的形式工作，也有一些GUI工具，包含了Git的部分功能,

Git Setup

Windows版本的Git可以直接在Git的官网上下载, 可以在PowerShell中输入git --version检查是否安装好Git了。

第一次安装Git时，需要配置Git的基本信息，最重要的是个人的身份信息，这将出现在之后的每一次提交之中。

Getting Help

可以使用git help <verb> 命令获得帮助。

亦可以git <command> -h 来获得帮助。

Ignoring Files

有时候程序会自动生成一些中间文件，这些一般不需要Git来保存版本，在目录下创建.gitignore文件，将想排除的文件写进去。

在https://github.com/github/gitignore有现成的排除文件。

Viewing Staged and Upstaged Changes

使用git diff命令，比较工作目录和暂存区文件的区别。

使用git diff --staged 命令，比较暂存区 和上次提交之间的文件的区别。

使用git difftool使用外部程序比较差异

Committing Changes

使用git commit -m "message"提交暂存区的修改。

使用git commit -a -m"message"来跳过暂存区，直接提交所有的修改。

Removing Files

先在工作目录内删除文件，再用git rm <file>暂存删除操作。

也可以从暂存区删除文件，但在文件目录中保留，也就是让Git不再最终该文件。用git rm --cached <file>。

Moving Files

使用git mv file_from file_to来移动文件。

Viewing the Commit History

用git log可以看提交的历史，有用的参数如下

-p 显示每次提交的补丁
--stat 显示每次提交的修改的文件的状态
--abbrev-commit 只显示前几个SHA-1字母
--relative-date 显示相对日期
--graph 显示图
--pretty oneline, short,full,fuller or format
--oneline --pretty=oneline --abbrev-commit 的简写
--decorate 显示HEAD指向哪个分支

Limiting Log Output

可以控制输出哪些提交的历史

Option Description
- 显示最近n条提交
--since, --after 指定日期之后
--until, --before 指定日期之前
--author 特定作者的提交
--comitter 特定人的提交
--grep 提交信息中包含特定字符串
-S 提交代码中包含特定字符串

--no-merges可以不显示合并历史

Undoing Things

想要修改最近的一次提交，包括文件或者提交信息，用git commit --amend。

Unstaging a Staged File

使用git reset HEAD <file> 可以撤销对文件的缓存

Unmodifying a Modified File

使用git checkout -- <file> 可以丢弃文件的修改，将文件回退到上一次提交的情况。

Working with Remotes

Git的一个强大的功能，可以与远程仓库协作。

Showing Remotes

命令git remote显示目前有的远程服务器的简写，可用-v显示对应的URL。

命令git remote show <remote> . 可以看到远程仓库的分支状态，分支是否被追踪，本地分支追踪哪个远程分支。

Adding Remote Repositories

可以用git remote add <shortname> <url> 来添加远程仓库。

Fetching and Pulling

命令git fetch <remote>可以从远程仓库取回所有本地没有的数据。

命令git pull可以从远程仓库取回数据并自动与本地分支合并。

Pushing to Remotes

命令git push <remote> <branch> 可以向远程仓库推送本地分支。

Renaming and Removing Remotes

命令git remote rename name_before name_after 可以更改远程仓库的名字。

命令git remote remove <remote> 可以删除远程仓库。

Tagging

Listing Tags

命令git tag可以列出所有的分支。 选项-l可以列出包含特定字符串的Tag。

Creating Tags

Git包含两种标签类型，annotated 和 lightweight. 如同名字所预示，annotated 包含作者、信息、日期等等信息，而lightweight则是一个简单的临时标签。

Annotated Tags

命令git tag -a v1.0 -m "My message" 可以创建标签。

命令git show v1.0可以显示标签信息。

命令git tag -a v1.0 SHA可以对编号为SHA的提交添加标签。

Lightweight Tags

命令git tag v1.0-lw可以创建轻量标签。

Sharing Tags

命令git push origin <tagname>可以向远程仓库推送标签。

Deleting Tags

命令git tag -d <tagname>可以删除本地的标签。

命令git push origin :ref/tags/<tagname>可以删除远程仓库的标签。

Checkout Tags

命令git checkout <tagname>可以检出标签状态，但此时仓库处于Detached HEAD状态，所有的提交都将不会保存。

Git Aliases

Git 别名可以让命令输入更快速简单

命令git config --global alias.co checkout 可以将git commit简化为git ci.

Git Branching

本章介绍Git中最有用的分支概念，Git鼓励开发者频繁使用分支。

Basic Branching and Mergin

Creating a New Branch

命令git branch testing将创建新的分支。

命令git log --decorate可以看目前处于哪个分支。

Switching Branch

命令git checkout testing将把HEAD指针指向testing分支，此时工作目录的文件状态将完全变成testing分支此时的状态。

此时提交修改将把HEAD对应的分支向前移动。

可用命令git checkout -b testing快速切换到新的分支。

切换前，如果工作区和暂存区有没有提交的修改，会提示切换失败，所以切换前需提交所有的修改，保证工作区是干净的。可以用Stashing 和 Cleaning 工具来保存工作区状态。

Basic Merging

如果目前有两个分支branch1,branch2需要合并，切换到branch1后，命令git merge branch2将把branch2合并到branch1中。

如果branch1恰好在branch2的后方，而不是有共同祖先但是分叉了，Git将用fast merge策略，不创造新的提交，而是直接将branch1的向前移动到branch2.

否则，如果两个分支没有修改同一个文件，Git会用recursive策略，创造一个新的提交，并将branch1指向这个新的提交。

如果两个分支修改了同一个文件，则需要手动处理冲突。

命令git status会列出冲突的文件，而且该文件中冲突的部分会被git直接列出来，如下:

当前的文件在======的上方，用来合并的修改在下方。可以用git mergetool调用外部的工具处理冲突。

修改后直接提交即可。

Branch Management

git branch可以看当前有多少分支，参数-v可以列出最近的提交。

参数--merged和--no-merged可以显示哪些分支被合并到当前分支中了。这些选项后可跟着其他的branch, 从而可以在当前分支中查看其他分支的情况。

合并之后可以用git branch -d <branch>删除分支，或者没有合并想直接删除的话用git branch -D <branch>。

Remote Branches

远程分支可以由git remote show [remote]显示。被跟踪的远程分支被命名为<remote>/<branch>。 可用git fetch <remote>抓取远程仓库的更新。

Pushing

命令git push <remote><branch>可以将本地分支推送到远程仓库中。

Tracking Branches

如果是远程仓库有而本地没有的分支，使用git fetch <remote>可以抓取远程仓库的更新到本地的远程仓库，然后用git checkout -b <branch> <remote>/<branch> 来创建对应的本地分支，或者用git merge <remote>/<branch>来合并到本地分支。

git checkout -b <branch> <remote>/<branch> 有一个简化版本git checkout --track <remote>/<branch>， 也有更简单的版本git checkout <branch>, 如果本地没有名为branch的分支，将自动从远程仓库创建一个。

如果已经有一个本地分支，想将其与远程分支关联起来，可以用git branch -u <remote>/<branch>。-u是--set-upstream-to的简写。

上游分支(upstream branch)可以用更方便的记号引用， 如@{u}或@{upstream}。

可以用git branch -vv来看哪些分支跟踪了什么。

Pulling

git pull 相当于git fetch和git merge。

Deleting Remote Branches

删除远程分支，需要用命令git push <remote> --delete <branch>

Rebasing

整合分支的另外一种方法称之为变基(rebase)。 回忆一下，合并是创造一个新的提交，形成一个类似环状的结构。当我们想有一个更清晰简单的提交历史，可以用rebase。

用git rebase --onto master server client 可以达成

Take the client branch, figure out the patches since it diverged from the server branch, and replay these patches in the client branch as if it was based directly off the master branch instead.

的效果。

用git rebase <basebranch> <topicbranch> 可以达到检查topic branch 并将其变基到master中的效果。

然后就可以git checkout master并快速前进git merge <topicbranch>，随后可以安全的删除掉topic branch.

注意，不要对上传到远程仓库的分支使用rebase, 因为其他人可能基于那个分支做了些工作。
